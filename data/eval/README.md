# Evaluation Artifacts

This directory owns committed runtime evaluation artifacts and local placement
policy for generated evaluation data.

The experimental default pointer is `default-artifact.json`. It resolves
`artifact_manifest` relative to this directory. The current default is:

```text
artifacts/pattern-v2-endgame-lite-100k-mt-v0/manifest.json
```

## Layout

```text
data/eval/
|-- README.md
|-- default-artifact.json
`-- artifacts/
    `-- pattern-v2-endgame-lite-100k-mt-v0/
        |-- weights.bin
        |-- manifest.json
        |-- provenance.json
        |-- README.md
        `-- NOTICE.md
```

`manifest.json` is the runtime loader contract. It records the format, pattern
set, score scale, relative `weights_file`, runtime checksum, phase count, phase
biases, and pattern table shape.

`provenance.json` records publication and validation metadata. Source
attribution, redistribution flags, trainer mode, validation summary, and
non-claims belong there rather than in the runtime manifest.

## Commit Rules

Only these files may be committed for promoted evaluation artifacts:

* `data/eval/default-artifact.json`
* `data/eval/README.md`
* `data/eval/artifacts/*/weights.bin`
* `data/eval/artifacts/*/manifest.json`
* `data/eval/artifacts/*/provenance.json`
* `data/eval/artifacts/*/README.md`
* `data/eval/artifacts/*/NOTICE.md`

Do not commit raw transcripts, normalized corpora, selected TSVs, teacher
labels, move-teacher cache files, local reports, trainer reports, generated
datasets, or logs. Local generated data belongs under `data/eval/local/` or
outside the repository.

The CTest target `vibe_othello_eval_artifact_commit_policy` validates the
committed artifact structure, runtime checksum, SHA-256 provenance, required
redistribution flags, and absence of local absolute paths.

## Runtime Behavior

The engine default resolver reads `data/eval/default-artifact.json`, resolves
the manifest relative to `data/eval/`, then resolves `weights_file` relative to
the manifest directory.

If the default pointer, manifest, weights file, runtime checksum, or binary
loader validation fails, the engine fails loudly. It does not silently fall
back to the static evaluator for a corrupt or missing default artifact.

Escape hatches:

* `--eval-artifact <manifest-path>` loads a custom artifact explicitly.
* `--eval-mode static` uses the legacy static evaluator and bypasses the
  default artifact.

## Source Boundary

This repository does not redistribute raw Egaroucid transcripts, normalized
corpora, selected TSVs, teacher labels, move-teacher TSVs, cache files, local
reports, or logs.

The committed v0 artifact was generated by the vibe-othello project using its
own exact solver and training pipeline. It is not an official Egaroucid
artifact.
